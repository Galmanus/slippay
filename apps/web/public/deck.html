01 · a verdade dura

A Stripe escreve
"limite de gasto".
Mas o limite mora
na base de dados dela.

  • Settlement em 2-7 dias — mandato regulatório de chargeback (Reg E nos EUA, PSD2 na UE). Stripe não pode encurtar sem virar money transmitter sem licença.
  • Piso de ~2.9% + $0.30 — interchange Visa é ~1.5% sozinho. Stripe é majoritariamente pass-through, não tem como descer.
  • Intermediário obrigatório — o merchant nunca recebe direto. Stripe pode congelar. Casos documentados de $130k congelados sem explicação.

Esses limites não são feature. São constitutivos do que faz a Stripe valer $70bi.

02 · o primitivo

O cliente assina
uma policy.
O bytecode aplica.

Quando o usuário toca o Face ID, uma struct Policy é gravada no smart wallet dele — no contrato Soroban, não num servidor.

// Soroban contract storage · per merchant struct Policy { merchant: Address, // quem pode cobrar (exato) max_per_charge: i128, // teto on-chain por ciclo interval_seconds: u64, // gap mínimo entre cobranças expires_at: u64, // auto-revoga após revoked: bool, // kill switch · só usuário }

A Slippay não pode mudar. O merchant não pode mudar. O validador não pode mudar. Só o usuário.

04 · ux do pix · enforcement do bytecode

Uma URL.
Um toque.
Cinco segundos.

  • Zero campos digitados. Sem email, senha, seed phrase, seleção de chain, modal de gas, checkbox de termos.
  • Passkey + Soroban smart wallet nativos do Stellar Protocol 21 (secp256r1 on-chain). Sem MetaMask, sem extensão. iCloud Keychain / Google Password Manager sincroniza entre devices.
  • Recurring sem novo toque. O merchant puxa via SAC.transfer; o smart wallet do cliente autoriza via policy match. Stripe-grade UX. Bytecode-grade enforcement.

benchmark: pix automático é 3 taps + biometria. nosso fluxo cobre o mesmo — cross-border, composable, fora do perímetro de PSP regulado.

05 · por que agora

Três forças
convergindo.

Macro

98% da cripto cross-border BR no Q1/2026 foi stablecoin. US$ 6,8bi num trimestre. +100% YoY. O brasileiro já se dolarizou — falta o trilho doméstico.

Regulação

Res. 561 do BCB ilegaliza gateway cross-border de stablecoin em out/2026. Mata os competidores que dependem de pernas exteriores. SlipPay é doméstico por design.

Tecnologia

Protocol 21 (jun/2024) adicionou secp256r1 nativo no Stellar. Passkey signa contrato direto. Fee ~$0.00001. Finalidade em 3-5s. O stack que destrava o produto agora existe.

06 · um trilho, dois mercados

Hoje os humanos.
Amanhã os agentes.

  • Hoje: SMBs e creators que a Stripe exclui ou onera (BR, LATAM, África). Recebem em dólar, em segundos, sem custódia. Zero chargeback overhang.
  • Amanhã: agentes de IA assinando API, dados, computação. Recurring de agente é território vazio — x402 resolveu one-shot, não assinatura. SlipPay já tem o contrato vivo.
  • Mesmo trilho. A mesma policy primitive que protege a humana protege a wallet do agente.

09 · pra onde vai

Não é billing.
É inteligência
de assinatura.

  • Stripe vê o pedido. SlipPay vê a wallet — saldo, gas, on-chain history. Prevê a falha antes dela acontecer.
  • SaaS perde 20-40% pra churn involuntário (Recurly · ProfitWell). Em cripto é pior. SlipPay avisa 3 dias antes, com a razão exata.
  • O agente propõe (pausa · downgrade · troca de ativo · top-up de gas) e o usuário aprova respeitando a policy. Proativo, não débito autônomo.
  • Cada merchant ganha um CFO de assinatura embutido — em mainnet, custo marginal perto de zero.
